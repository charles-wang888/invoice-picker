# 发票凑数神器 - 演示步骤

本文档详细描述「发票凑数神器」Web 应用的完整演示流程，包括环境准备、数据生成、启动应用、执行凑单、查看结果与导出。

---

## 一、环境准备

### 1.1 确认 Python 版本

需 Python 3.8 及以上：

```bash
python --version
```

### 1.2 安装依赖

在项目根目录下执行：

```bash
pip install -r requirements.txt
```

或单独安装 Streamlit：

```bash
pip install streamlit
```

### 1.3 验证内核

```bash
python -c "from invoice_selector import select_invoices; print('OK')"
```

若输出 `OK`，说明内核安装正常。

---

## 二、生成原始发票数据

### 2.1 运行生成脚本

```bash
python generate_raw_data.py
```

### 2.2 生成结果

- 输出文件：`raw_data.txt`
- 数量：10000 张发票
- 面额：以 100 块、1000 块左右为主，少量 2000 块档（便于凑 40 万等较小目标）
- 格式：每行「20 位发票号码 + 空格 + 金额」，金额保留 2 位小数
- 顺序：随机打乱，不按金额排序

示例前几行：

```
04400000000000004935 161.53
04400000000000009945 796.90
04400000000000001237 1395.41
...
```

---

## 三、启动 Web 应用

### 3.1 启动方式

方式一（推荐）：

```bash
python main.py
```

方式二：

```bash
streamlit run main.py
```

### 3.2 打开页面

启动成功后，终端会显示本地地址（通常为 `http://localhost:8501`），浏览器会自动打开；若未自动打开，可手动在浏览器中输入该地址。

### 3.3 页面布局

- 标题：**发票凑数神器**
- 上方：文件上传区、要选出的发票张数 Y、要凑的总金额 Z
- 下方：执行按钮、重新计算按钮（有缓存时显示）

---

## 四、执行凑单演示

### 4.1 准备并上传发票文件

1. **按格式准备真实数据**：参考 `raw_data.txt` 的格式，每一行表示一张发票，包含 2 个字段：**字段 1** 为发票编号，**字段 2** 为发票金额（两字段之间用空格分隔；金额建议保留 2 位小数）。
2. **上传数据文件**：在「上传发票数据文件」区域点击「Browse files」，选择已准备好的数据文件（如 `.txt`），确认文件名出现在上传区即表示已选中。

### 4.2 设置参数

**要选出的发票张数 Y：**

- 数值输入框，默认 1000
- 可改为任意正整数，但须 ≤ 发票总数

**要凑的总金额 Z（元）：**

- 文本输入框，支持整数或最多两位小数
- 示例：`432245`、`432245.5`、`1000000`
- 若格式错误（如三位小数、非数字），输入框下方会显示红色提示

### 4.3 点击「执行」

1. 点击蓝色「执行」按钮
2. 页面会显示加载进度：「正在计算 Top 3 解法（满轮数 + 不同 seed）...」
3. 计算完成后，下方会展示「解法（共 N 种不同组合）」及整体运行时长

### 4.4 演示场景示例

| 场景 | Y   | Z       | 说明                         |
|------|-----|---------|------------------------------|
| 凑 40 万 | 1000 | 432245  | 小目标，适合验证小面额分布   |
| 凑 100 万 | 1000 | 1000000 | 常规场景                     |
| 少张凑大额 | 200  | 500000  | 少量发票凑较大金额           |

## 五、查看结果

### 5.1 解法概览

- 每个解法以**可展开/收起的区块**展示
- 区块标题示例：**解法 1**：汇总 432,350.25 元，误差率 **0.0243%**
- 默认展开「解法 1」，其余可点击展开

### 5.2 每个解法包含的信息

- **汇总金额**：选中 Y 张发票的总金额
- **误差率**：(汇总 − Z) / Z × 100%
- **选中张数**：必为 Y 张

### 5.3 凑数发票清单

- 表格列出该解法下**全部 Y 张**发票的「发票编号」「面额（元）」
- **分页显示**：每页 20 条，支持上一页、下一页及页码快速跳转（1、2、3…）
- 1000 张发票为 50 页，可通过页码区快速定位

### 5.4 组合去重说明

- 若 3 次运行得到相同组合，仅展示 1 种
- 若 2 种相同、1 种不同，则展示 2 种
- 标题会显示：「解法（共 N 种不同组合）」

## 六、导出 CSV

### 6.1 导出按钮

在每个解法展开区内，表格上方有按钮：**导出 CSV（完整清单）**。

### 6.2 点击导出

1. 点击「导出 CSV（完整清单）」
2. 浏览器会下载一个 CSV 文件
3. 默认文件名示例：`发票凑数_解法1_Y1000_Z432245.00.csv`

### 6.3 CSV 内容

- 编码：UTF-8 with BOM，Excel 可直接打开不乱码
- 列：发票编号、面额（元）、目标金额Z、汇总金额、误差率%
- 行数：该解法下**全部 Y 张**发票，非当前页 20 条

---

## 七、重新计算与参数变更

### 7.1 修改参数后重新执行

1. 修改「要选出的发票张数 Y」或「要凑的总金额 Z」
2. 再次点击「执行」
3. 系统会用**新参数**重新计算，并**覆盖**此前缓存结果

### 7.2 使用「重新计算」按钮

- 当已存在缓存结果时，会出现「重新计算」按钮
- 点击后清空缓存，可重新上传文件、修改参数后再次执行

### 7.3 更换文件

- 若需使用新的 `raw_data.txt` 或其他发票文件，重新选择文件后点击「执行」即可
- 注意：Streamlit 在部分交互后可能清空文件选择，此时需重新选择文件

---

## 八、常见问题与操作提示

| 现象 | 处理方式 |
|------|----------|
| 显示「请先上传发票数据文件」 | 选择 `raw_data.txt` 后再点击「执行」 |
| 显示「格式错误」 | 检查 Z 是否为整数或最多两位小数，且大于 0 |
| 显示「未能得到可行组合」 | 目标 Z 过高或可选发票不足，可减小 Z 或检查数据 |
| 分页点击「下一页」后内容消失 | 已修复：点击「执行」后会缓存结果，翻页会正常展示 |
| 误差率很高（如 100%+） | 检查是否用新 Z 重新计算；若目标与初始和差距大，系统会自动增加交换轮数 |

## 九、演示流程总结

1. **准备**：安装依赖 → 运行 `generate_raw_data.py` 生成 `raw_data.txt`
2. **启动**：运行 `python main.py` → 浏览器打开 Web 页面
3. **输入**：上传 `raw_data.txt` → 设置 Y、Z
4. **执行**：点击「执行」→ 等待计算完成
5. **查看**：展开各解法 → 浏览分页清单 → 查看汇总金额与误差率
6. **导出**：点击「导出 CSV（完整清单）」→ 保存并打开 CSV

完成上述步骤即可完整演示发票凑数神器的全部功能。
